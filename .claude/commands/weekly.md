# /weekly 工作流

执行本周 SSIR 中文版编辑部的完整流程。我会按以下顺序推进，每个关键节点会停下来与你确认。

## 阶段 1：抓取候选

调用 `@ssir-scraper` 获取本周 SSIR 新发表的文章元数据。

它会输出到 `data/articles/_index_{YYYY-WW}.json`。

完成后，我会把候选列表（标题 + 一句话摘要）展示给你。

## 阶段 2：候选评估（可选）

如果候选 > 5 篇，我会基于 `prompts/select.md` 给每篇打分，并推荐 2-3 篇做编译。

⚠️ 这步只是建议，最终选哪几篇**由你决定**。

## 阶段 3：⏸ 人工介入

由于 SSIR 是订阅制，我无法直接抓取全文。请你：

1. 从你的 SSIR 订阅 / 用 Claude in Chrome 浏览器扩展打开选中的文章
2. 把**每篇文章的完整正文**粘贴回这里（一篇一篇来），每篇我都会调用 `@article-curator` 整理存档

格式建议：
```
[文章 1 的 URL]
[粘贴完整正文]
---
[文章 2 的 URL]
[粘贴完整正文]
```

## 阶段 4：双轨编译（自动）

对每篇人工提供的文章：

- 调用 `@wechat-editor` → `data/output/{week}/{slug}/wechat.md`
- 调用 `@xhs-writer` → `data/output/{week}/{slug}/xhs.json`

两个 agent 互不依赖，可以并行触发。

## 阶段 5：配图（自动）

对每篇小红书产出，调用 `@xhs-image-designer` 生成封面 + 图卡，存到 `data/output/{week}/{slug}/images/`。

## 阶段 6：汇总报告

生成 `data/output/{week}/summary.md`，列出：

- 本周处理了哪几篇
- 各篇产出文件清单
- 风险点 / 新增术语 / 译者注汇总
- 待人审项

完成后我会把 summary 路径告诉你。

---

**开始之前**：请确认本次想跑全流程还是某一段？

- 完整跑（推荐周一早上）：直接回复"开始"
- 只跑抓取（看看本周有什么）：回复"只抓取"
- 跳过抓取，直接编译已有文章：回复"编译 {slug}" 或直接调用对应 sub-agent
